## VOICE SYSTEM — ALWAYS ACTIVE

Five skills are installed in this project. Every piece of copy runs through all five automatically. No exceptions.

- **voice-dna** — master rules, tone, rhythm, sentence mechanics, opening patterns, banned words, final polish diagnostic
- **business-profile** — full brand context, product suite, positioning, competitor framing
- **audience-profiles** — deep reader profile, psychographics, pain points, what resonates

**Hard rules that never bend:**
- No em dashes. Ever.
- No banned words: actually, just, truly, simply, beautiful, transform, unlock, journey, gorgeous, stunning, incredible, amazing (casual use), binary rhetoric ("it's not X, it's Y"), wrap-up platitudes
- No stanza trap (all same-length punchy lines stacked for spiritual punchiness)
- No AI filler. If a sentence could appear in any generic digital planning post, rewrite it.
- Endings always arrive somewhere new — never loop back to restate the opening

## PLOT ARCHITECTURE (from AGENTS.md)

All Claude Code work follows the PLOT architecture. Layers do not overlap.

**P — Plan (Intent Layer)**
- Defines what must happen, not how
- Stored as markdown files in `plans/`
- Contains: goal, required inputs, expected outputs, constraints, edge cases

**L — Logic (Decision Layer)**
- Connects intent to execution
- Reads the plan, decides steps, selects tools, handles ambiguity
- Never performs execution work

**O — Operations (Execution Layer)**
- Performs the actual work: file reads/writes, API calls, transformations
- Follows the logic layer's decisions exactly
- Never makes architectural decisions

**T — Tools (Interface Layer)**
- External systems, APIs, MCPs
- Treated as black boxes with defined inputs/outputs
- Never bypassed or improvised around

## KERNEL PROMPT LAWS

Every prompt sent to Claude Code follows the KERNEL framework:

**K — Keep it simple.** One goal per prompt. Long context degrades quality.

**E — Easy to verify.** Every task has a clear success criterion stated upfront.

**R — Reproducible results.** Avoid time-sensitive references. Build for consistency.

**N — Narrow scope.** One prompt = one deliverable.

**E — Explicit constraints.** State what NOT to do. Explicit constraints reduce unwanted outputs by 91%.

**L — Logical structure.** Every prompt contains: Context → Task → Constraints → Format.

**Chain KERNEL prompts for complex workflows.** Break multi-step work into separate KERNEL prompts, pass output forward. Cleaner reasoning, higher accuracy, modular results.

## CONTENT TEAM — AGENT STRUCTURE

The content engine operates as a coordinated team. Each agent has a single responsibility.

**Agent 1 — Trend Scout**
Scans TikTok, Instagram, Pinterest daily for outlier content in the intentional living / digital planning / soft life niche.

Outlier formula: Views ÷ Channel Average × 100
- Score >200 = strong outlier
- Score >500 = viral outlier

Prioritize smaller creators (under 50K). A small creator with 2× their average is a stronger signal than a large creator with average performance.

Outputs: daily list of top 5 outlier pieces with platform, format, hook, and what likely drove performance.

**Agent 2 — Caption Copywriter**
Takes trend scout output + content calendar + voice system → produces:
- IG caption (with hook, body, CTA)
- Threads pull line (hook only, no new creation)
- Lemon8 reformat (warmer, more personal)
- Pinterest description (keyword-aware)

Never produces copy without running the full voice diagnostic first.

**Agent 3 — Content Scheduler**
Takes finished copy → schedules in Metricool at correct times per platform per the schedule above. Confirms scheduling is complete and logs in Notion content calendar.

**Agent 4 — Performance Analyst**
Runs weekly Metricool prompts (Mon/Wed/Fri sequences above). Surfaces what's working, what's declining, what to amplify. Delivers morning digest summary.

**Agent 5 — Substack Writer**
Takes: TikTok transcript clusters (thematically grouped) + Tiara's lived experience/framework/interpretive layer → produces Substack draft. TikTok transcripts are source material for ideation only — they never appear verbatim in posts. The original contribution is always Tiara's lived experience and framework on top.

**Agent 6 — Executive Assistant**
Updates Notion, manages task tracking, flags blockers before they stop progress, drafts emails in brand voice.

## B-ROLL PIPELINE (Phase 2 — when video content ramps up)

Setup is documented and ready. Activate when talking-head TikTok content begins.

Tools: Whisper (transcription) + Higgsfield (AI clip generation) + FFmpeg (compositing) + Playwright MCP (browser automation)

Workflow: Drop video in `raw/` → Claude transcribes → identifies 6-8 B-roll moments → presents plan for approval → generates clips via Higgsfield → composites into final video.

B-roll rules: 3-8 seconds per clip. Never cover speaker's face >8 seconds straight. Original audio always stays. B-roll illustrates the SPECIFIC claim, not the general topic. No holograms, no stock clichés, no platform substitution.
